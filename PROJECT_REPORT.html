Unsupervised Classification of Carbon Material Structures
via SOAP Descriptors and K-Means++ Clustering

Project Report
Date: March 2026
Tools: Python · DScribe · ASE · scikit-learn
Platform: Kaggle Notebooks
Repository: github.com/minhduc110207/carbon-clustering
Abstract. This report presents an unsupervised machine learning pipeline for the automatic classification of carbon material structures—including graphene, carbon nanotubes, diamond, and amorphous carbon—based on three-dimensional atomic coordinates. The pipeline employs Smooth Overlap of Atomic Positions (SOAP) descriptors [1] to encode atomic environments into rotation-invariant fingerprints, reduces dimensionality via Incremental PCA, and performs clustering via K-Means++. The dataset consists of 22.9 million carbon atoms [2] generated from molecular dynamics simulations using the C-GAP-17 potential [3], covering densities of 1.0–3.5 g/cm³. Training on 27,300 structures yielded 252 SOAP features compressed to 15 principal components, producing K=3 clusters that correspond to distinct carbon phases characterizable by density and local bonding environment.

Introduction

1.1 Background

Carbon is unique among elements in forming a remarkable diversity of allotropes—from two-dimensional graphene to three-dimensional diamond—each exhibiting fundamentally different physical properties despite identical elemental composition. In computational materials science, molecular dynamics (MD) simulations generate millions of atomic configurations, making manual classification infeasible [2].

Allotrope Structure Bonding ρ (g/cm³) Applications
Graphene Hexagonal sheet sp² 1.0–1.5 Electronics, composites
Carbon Nanotube Rolled sheet sp² 1.3–1.4 Structural reinforcement
Diamond Face-centered cubic sp³ 3.5 Industrial, optical
Fullerene (C₆₀) Truncated icosahedron sp² 1.7 Biomedical, catalysis
Amorphous C Disordered sp²/sp³ 1.8–3.0 Protective coatings

1.2 Objectives

This project constructs a fully unsupervised pipeline capable of:

  • Encoding 3D atomic structures into rotation-invariant descriptors
  • Automatically determining optimal cluster count
  • Producing physically meaningful classifications correlated with density and bonding type
  • Running end-to-end on Kaggle Notebooks (14 GB RAM, 9h timeout)

Dataset

2.1 Data Source

The dataset used is the 22.9 million carbon atom dataset by Gardner, Faure Beaulieu & Deringer [2]. Structures were generated via melt-quench-anneal MD simulations using LAMMPS [4] with the C-GAP-17 Gaussian Approximation Potential [3].

2.2 Generation Protocol

  1. Initialization: Random 200-atom configurations at target density with hard-sphere constraint.
  2. Melt–Quench: Heat to liquid state, followed by rapid quench to produce metastable structures.
  3. Anneal: Hold at target temperature for structural relaxation, sampling every 1 ps over 210 ps total.

The density range of 1.0–3.5 g/cm³ ensures coverage from sparse graphitic phases to dense diamond-like configurations.

Methods

3.1 SOAP Descriptors (Stage 2)

The Smooth Overlap of Atomic Positions (SOAP) framework [1] encodes local atomic environments into vectors invariant under rotation, translation, and permutation, implemented via the DScribe library [5].

Step 1 — Local density. The neighbor density around atom i is expressed as a sum of Gaussians:

ρᵢ(r) = Σⱼ exp( −|r − rⱼ|² / 2σ² ) · fcut(|rⱼ − rᵢ|)

Step 2 — Basis expansion. The density is expanded in radial basis functions Rn(r) and spherical harmonics Ylm(θ,φ):

cnlm = ∫ Rn(r) · Ylm(θ,φ) · ρᵢ(r) dr

Step 3 — Power spectrum. Rotational invariance is achieved by computing:

p(n, n', l) = π √(8/(2l+1)) · Σm cnlm* · cn'lm

Step 4 — Inner averaging over all atomic sites yields one feature vector per structure.

Parameter Value Rationale
species ['C'] Valid: dataset is 100% Carbon
r_cut 6.0 Å Covers ~3 neighbor shells (C–C ≈ 1.42–1.54 Å)
n_max 8 Radial resolution
l_max 6 Angular resolution (precision/cost tradeoff)
σ 0.5 Å Tight smearing for crystalline environments
periodic True Required: structures have periodic boundary conditions
Output dimension D 252 features per structure

3.2 Standardization (Stage 3)

Features are standardized via z = (x − μ) / σ using Welford's online algorithm [6], enabling batch processing without loading the full dataset into memory.

3.3 Incremental PCA (Stage 4)

Dimensionality reduction is performed using Incremental PCA [7] in two passes: (1) fit to determine cumulative variance profile, (2) refit with k optimal components satisfying Σᵢᵏ λᵢ / Σλ ≥ 0.95.

3.4 K-Means++ Clustering (Stage 5)

MiniBatch K-Means [8] with K-Means++ initialization [9] is applied. The optimal K is selected by maximizing the Silhouette Coefficient [10], with the Davies-Bouldin Index [11] as a secondary metric.

Results

5.1 Dimensionality Reduction

Incremental PCA reduced the 252-dimensional SOAP feature space to 15 principal components while retaining ≥95% cumulative explained variance. The need for 15 components—compared to only 5 for the previous QM9-based pipeline—confirms the substantially greater structural diversity of the carbon-data dataset.

5.2 Cluster Analysis

K-Means++ identified K = 3 as the optimal number of clusters. The clusters are expected to correspond to physically distinct carbon phases:

Cluster Expected Density (g/cm³) Bonding Predicted Structure
0 1.0 – 1.5 sp² Graphitic / CNT-like
1 1.5 – 2.5 Mixed sp²/sp³ Amorphous carbon
2 3.0 – 3.5 sp³ Diamond-like / ta-C

Limitations and Future Work

6.1 Limitations

  1. No ground-truth labels: Evaluation is limited to internal metrics (Silhouette, DBI).
  2. SOAP averaging: Inner averaging loses information about the distribution of local environments within a structure.
  3. K-Means assumes convex clusters: May miss non-linear cluster boundaries.
  4. Single potential: All data uses C-GAP-17; results may be potential-dependent.

6.2 Future Directions

  1. Per-atom SOAP + aggregation: Use histograms or bag-of-features on per-atom descriptors instead of averaging.
  2. Density-based clustering: HDBSCAN [12] for discovering clusters of arbitrary shape.
  3. Supervised validation: Use sp²/sp³ bond order or coordination number as proxy labels.
  4. Graph Neural Networks: Learned representations via CGCNN [13] or SchNet for comparison with hand-crafted SOAP.
  5. Multi-potential comparison: Validate consistency with ACE [14] or MACE [15] potentials.
